Updated: 2026-01-31

## 직원 테이블 설계 요구사항

### 직원 유형 구분 (employee_type)
- **정규직**: 4대보험 전체 가입, 월급제
- **계약직**: 4대보험 전체 가입, 월급제
- **알바 (주 15시간 이상)**: 4대보험 전체 가입, 시급/일급제
- **알바 (주 15시간 미만)**: 산재보험만 의무, 시급/일급제 → 자체 DB 등록 필수

### 필수 필드
- `id`: UUID (PK)
- `user_id`: 사업자 계정 참조 (FK)
- `name`: 직원 이름
- `employee_type`: enum (정규직, 계약직, 알바)
- `weekly_hours`: 주간 근무시간 (알바의 경우 4대보험 적용 여부 판단 기준)
- `hourly_rate`: 시급 (알바용)
- `monthly_salary`: 월급 (정규직/계약직용)
- `insurance_status`: 4대보험 가입 상태 (자동연동/수동등록/미가입)
- `start_date`: 입사일
- `status`: 재직 상태 (재직/퇴사)
- `created_at`, `updated_at`: 타임스탬프

### 데이터 연동 전략
- **코드에프 연동 가능**: 4대보험 가입자 (정규직, 계약직, 주 15시간+ 알바)
- **자체 DB 필수**: 주 15시간 미만 단기 알바 (산재보험만 가입)
- **동기화 방식**: 코드에프 4대보험 데이터와 자체 DB 병합하여 전체 직원 현황 관리
