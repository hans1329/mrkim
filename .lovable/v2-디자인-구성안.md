## 2. 컬러 시스템

### 2.1 기상도 연동 동적 팔레트

V2의 핵심 차별점: 앱 전체 분위기가 세무 기상도에 따라 변함.

```css
/* ☀️ 맑음 — 따뜻한 낮 */
[data-weather="sunny"] {
  --v2-bg: 210 40% 98%;           /* 밝은 하늘색 배경 */
  --v2-surface: 0 0% 100%;       /* 순백 카드 */
  --v2-accent: 38 100% 55%;      /* 골드 포인트 */
  --v2-text: 220 15% 20%;        /* 진한 네이비 텍스트 */
  --v2-mood-gradient: linear-gradient(180deg, #E8F4FD 0%, #FFF9E6 100%);
}

/* 🌤️ 구름 조금 — 주의 */
[data-weather="partly-cloudy"] {
  --v2-bg: 45 30% 96%;
  --v2-surface: 40 20% 99%;
  --v2-accent: 38 90% 50%;       /* 앰버 */
  --v2-text: 220 15% 25%;
  --v2-mood-gradient: linear-gradient(180deg, #FFF8E1 0%, #F5F5F5 100%);
}

/* ⛅ 흐림 — 경고 */
[data-weather="cloudy"] {
  --v2-bg: 220 15% 94%;
  --v2-surface: 220 10% 98%;
  --v2-accent: 220 60% 55%;
  --v2-text: 220 15% 30%;
  --v2-mood-gradient: linear-gradient(180deg, #ECEFF1 0%, #E0E0E0 100%);
}

/* 🌧️ 비 — 위험 */
[data-weather="rainy"] {
  --v2-bg: 220 25% 90%;
  --v2-surface: 220 15% 96%;
  --v2-accent: 0 70% 55%;        /* 레드 경고 */
  --v2-text: 220 20% 20%;
  --v2-mood-gradient: linear-gradient(180deg, #CFD8DC 0%, #B0BEC5 100%);
}

/* ⛈️ 폭풍 — 긴급 */
[data-weather="storm"] {
  --v2-bg: 0 20% 92%;
  --v2-surface: 0 10% 97%;
  --v2-accent: 0 84% 55%;        /* 강렬한 레드 */
  --v2-text: 0 10% 15%;
  --v2-mood-gradient: linear-gradient(180deg, #FFEBEE 0%, #FFCDD2 100%);
}
```

### 2.2 시맨틱 컬러 (기상도 독립)

```css

  /* 보이스 버블 */

  --v2-voice-active: 210 100% 55%;
```

## 3. 타이포그래피

### 3.1 폰트 스케일

Big Typography 원칙: **핵심 숫자는 멀리서도 보여야 한다.**

```
시스템: Pretendard (한글) + Inter (영문/숫자)

--v2-text-hero:    72px / 1.0  / 800  → 매출 금액 (풀스크린)
--v2-text-display: 48px / 1.1  / 700  → 피드 내 핵심 숫자
--v2-text-title:   24px / 1.3  / 700  → 카드 제목
--v2-text-body:    16px / 1.5  / 400  → 본문 텍스트
--v2-text-caption: 13px / 1.4  / 400  → 보조 텍스트
--v2-text-micro:   11px / 1.3  / 500  → 칩, 배지
```

### 3.2 숫자 표현 규칙

- 금액: 항상 `--v2-text-display` 이상 사용
- 퍼센트 변화: 숫자 옆에 화살표 + 컬러 (↑ 초록, ↓ 빨강)
- 단위: 숫자보다 작은 사이즈 (`--v2-text-body`)로 붙임

```
127만원    ← "127" = 48px bold, "만원" = 16px regular
+23%↑      ← "+23%" = 24px bold 초록, "↑" = 아이콘
```

## 4. 컴포넌트 디자인

### 4.1 기상도 앵커

```
┌─────────────────────────────────┐
│  ☀️  오늘 매장 날씨: 맑음   👤   │  높이: 64px
│  ████████████░░░░ 절세 78%      │  게이지: 6px height
└─────────────────────────────────┘

• 배경: var(--v2-mood-gradient)
• 날씨 아이콘: 32px, 미세 애니메이션 (해 회전, 비 낙하 등)
• 절세 게이지: rounded-full, 초록→골드 그라디언트
• 아바타(👤): 28px 원형, 탭→설정 슬라이드
• 터치 시: 아래로 펼쳐지며 3축 상세 표시 (300ms spring)
```

### 4.2 미처리 액션 카드

```
┌─────────────────────────────────┐
│  ⚡ 미처리 2건                    │
│  ┌──────────┐ ┌──────────┐      │
│  │부가세 D-3│ │영수증 5건│      │
│  │ 🔴 긴급  │ │ 🟡 주의  │      │
│  └──────────┘ └──────────┘      │
└─────────────────────────────────┘

• 수평 스크롤 가능한 칩 카드
• 좌 스와이프: "나중에" (회색으로 변하며 축소)
• 우 스와이프: "처리하기" (해당 플로우 진입)
• 긴급도 테두리: 2px solid, 색상 코드별 구분
• 전체 0건이면 영역 자체 숨김 + 축하 메시지
```

### 4.3 피드 카드 (말풍선형)

```
  ┌─ 김비서 아바타 (32px)
  │
  │  ┌────────────────────────┐
  │  │ 사장님, 어제 카드매출이  │
  │  │                        │
  │  │    127만원              │  ← Big Typography
  │  │                        │
  │  │ 전주보다 +23% ↑ 🔥     │
  │  │                        │
  │  │ ───────────────────    │
  │  │ 📊 상세 보기            │  ← 인라인 액션
  │  └────────────────────────┘
  │       5분 전

• 모서리: 20px (좌상단만 4px → 말풍선 꼬리 느낌)
• 배경: var(--v2-bubble-bg)
• 그림자: 0 2px 12px hsl(var(--v2-bubble-shadow))
• 등장 애니메이션: translateY(20px) + opacity 0→1, 400ms spring
• 카드 간 간격: 16px
```

### 4.4 보이스 버블

```
         ┌─────┐
         │ 🎙️  │   56px 원형
         └─────┘
    [영수증] [매출] [세금]   ← 컨텍스트 칩

• Idle: var(--v2-voice-idle), subtle pulse (3s)
• 탭 시: 텍스트 입력 시트 올라옴
• 홀드 시: 원 확대(1.3x) + 파형 UI + var(--v2-voice-active)
• 음성 인식 중: 파형 애니메이션 + var(--v2-voice-listening)
• 발화 중: 원 주변 웨이브 링 확산
```

### 4.5 컨텍스트 칩

```
• 크기: height 32px, padding 0 12px
• 배경: hsl(var(--v2-surface))
• 테두리: 1px solid hsl(var(--v2-text) / 0.1)
• 모서리: rounded-full
• 텍스트: --v2-text-micro, 500
• 아이콘: 14px, 왼쪽
• 탭 피드백: scale(0.95) + 배경 어두워짐
• 수평 스크롤, no-scrollbar
```

### 4.6 딥워크 오버레이

```
┌─────────────────────────────┐
│  ← 거래 내역 정리        ✕  │  ← 헤더
├─────────────────────────────┤
│                             │
│  (기존 V1 뷰 재활용)         │  ← 콘텐츠
│                             │
└─────────────────────────────┘

• 진입: 하단에서 위로 슬라이드 (500ms spring)
• 배경: 피드 위 80% 높이, 뒤로 blur + dim
• 닫기: 하단 스와이프 or X 버튼
• 모서리: 상단만 24px radius
```

## 5. 모션 & 애니메이션

### 5.1 모션 원칙

```
1. 의미 있는 모션만: 장식 아닌 피드백 목적
2. 스프링 기반: 자연스러운 물리 느낌 (framer-motion)
3. 60fps 보장: transform + opacity 위주
4. 접근성: prefers-reduced-motion 존중
```

### 5.2 구체적 모션 명세

| 요소 | 트리거 | 모션 | 듀레이션 |
|------|--------|------|----------|
| 피드 카드 등장 | 스크롤 진입 | translateY(20) → 0 + fade | 400ms spring |
| 기상도 날씨 변경 | 데이터 갱신 | 아이콘 morph + 배경 컬러 전환 | 800ms ease |
| 절세 게이지 증가 | 미션 완료 | 게이지 fill + 숫자 카운트업 | 600ms |
| 미처리 스와이프 | 우 스와이프 | 카드 오른쪽 이탈 + 축하 파티클 | 300ms |
| 축하 이벤트 | 최고 매출 등 | 컨페티(폭죽) 파티클 | 2000ms |
| 보이스 활성 | 홀드 | scale(1→1.3) + 파형 시작 | 200ms spring |
| 보이스 파형 | 음성 인식 중 | 실시간 음량 반영 바 애니메이션 | 실시간 |
| 딥워크 진입 | 칩/음성 | translateY(100%) → 0 | 500ms spring |
| 프로필 슬라이드 | 아바타 탭 | translateX(100%) → 0 | 400ms spring |

### 5.3 축하 파티클 (컨페티)

```
트리거 조건:
- 미처리 0건 달성
- 이번 주/월 최고 매출
- 절세 게이지 100%
- 세금 환급금 발생

구현: canvas-confetti 또는 커스텀 CSS 파티클
지속: 2초 후 자동 소멸
```

## 6. 반응형 전략

### 6.1 모바일 (< 768px) — 기본

- 단일 컬럼 피드
- 보이스 버블 하단 고정
- 풀스크린 오버레이

### 6.2 태블릿 (768-1024px)

- 피드 카드 폭 최대 480px, 중앙 정렬
- 딥워크 오버레이 80% 폭

### 6.3 PC (> 1024px) — 3패널

```
┌────────────┬──────────────────┬────────────┐
│  280px     │    480px 중앙    │   flex-1   │
│            │                  │            │
│ 대화       │  기상도 + 피드    │  상세 패널  │
│ 히스토리   │  (모바일 동일)    │  (카드 클릭│
│            │                  │   시 표시) │
│            │                  │            │
│            │  보이스 버블      │            │
└────────────┴──────────────────┴────────────┘
```

- 좌측: 이전 대화 히스토리 (선택하면 피드에 관련 카드 표시)
- 우측: 기본은 빈 패널 "카드를 선택하세요" → 클릭 시 상세

## 7. 접근성 & 성능

### 접근성
- 모든 인터랙티브 요소에 aria-label
- 기상도 색상에만 의존하지 않고 텍스트 라벨 병행
- prefers-reduced-motion 시 모션 최소화
- 최소 터치 타겟 44x44px

### 성능
- 피드 카드: 가상 스크롤 (react-window) 적용
- 이미지/차트: lazy loading + skeleton
- 모션: GPU 가속 (transform, opacity만 사용)
- 첫 화면: 기상도 + 최근 피드 3장만 초기 로드

## 9. V1 디자인 토큰과의 공존

V2 토큰은 `[data-theme="v2"]` 스코프 안에서만 적용.
기존 V1 페이지는 전혀 영향 없음.

```css
/* index.css에 추가 */
[data-theme="v2"] {
  /* V2 전용 토큰 (기상도별 동적 값은 JS로 data-weather 설정) */
  --v2-bg: 210 40% 98%;
  --v2-surface: 0 0% 100%;
  /* ... */
}
```

`/v2` 라우트 진입 시 `<div data-theme="v2" data-weather="sunny">` 래퍼 적용.
