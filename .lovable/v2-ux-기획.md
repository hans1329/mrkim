# 김비서 V2 UX 기획서: "1화면 비서"

> 최종 수정: 2026-04-14

---

## 1. 핵심 철학

> **"앱을 쓰는 게 아니라, 비서와 만나는 것"**
>
> 메뉴 없음. 검색 없음. 탭 없음.
> 열면 김비서가 오늘 할 일을 알려주고, 말로 끝낸다.

### 디자인 원칙
- **No-Touch**: 손이 비어있지 않은 소상공인을 위한 음성 우선 인터페이스
- **No-Think**: 판단을 요구하지 않고, 김비서가 먼저 제안하고 실행
- **No-Navigate**: 탭/메뉴 없이 피드와 보이스만으로 모든 기능 접근

---

## 2. 메인 인터랙션 패러다임: 피드 + 보이스

인스타그램 피드처럼 김비서가 먼저 말을 거는 카드를 시간순 스크롤하며,
보이스 버블로 즉시 음성 대화를 시작할 수 있는 구조.

---

```
┌─────────────────────────────┐

├─────────────────────────────┤
│ ⚡ 미처리 액션 (Sticky)       │ ← ② 고정: 처리 시 축하 모션
├─────────────────────────────┤
│                             │
│ 💬 김비서 피드 (스크롤)       │ ← ③ 메인 콘텐츠
│    매출요약 / 인사이트 /      │
│    영수증촬영 / 축하 카드     │
│                             │
├─────────────────────────────┤
```

## 4. 영역별 상세 기획

### ① 세무 기상도 앵커 (상단 고정)

날씨 메타포로 현재 세무/재무 상태를 직관적으로 전달.

| 상태 | 아이콘 | 배경 톤 | 트리거 조건 |
|------|--------|---------|------------|
| ☀️ 맑음 | 해 | 밝은 블루/화이트 | 미납 0, 매출 정상, 마감 여유 |
| 🌤️ 구름 조금 | 구름+해 | 연한 옐로우 | 마감 7일 이내 or 경비 증가 |
| ⛅ 흐림 | 구름 | 회색톤 | 미분류 거래 다수 or 마감 3일 |
| 🌧️ 비 | 비 | 어두운 블루 | 세금 폭탄 예상 or 미납 존재 |
| ⛈️ 폭풍 | 번개 | 레드톤 | 마감일 당일 + 미처리 |

**3축 상세 (터치 시 펼침)**:
```
☀️ 맑음
├── 매출 흐름: 좋음 ✅
├── 세금 리스크: 주의 ⚠️
└── 증빙 상태: 양호 ✅
```

**절세 게이지**: 김비서와 미션(영수증 촬영, 분류 확인 등) 완료 시 차오름 → 게이미피케이션

---

### ② 미처리 액션 (Sticky 영역)

- 피드와 별개로 기상도 바로 아래에 고정
- 처리할 때까지 사라지지 않음
- 긴급도에 따라 테두리 색상 변화 (🔴→🟡→🟢)
- **스와이프 인터랙션**: 오른쪽 → 지금 처리 / 왼쪽 → 나중에
- 처리 완료 시 축하 모션 + 절세 게이지 증가

---

### ③ 김비서 피드

시간순 카드가 위에서 아래로 쌓이는 인스타 스토리형 피드.
김비서가 먼저 말을 거는 형태.

**카드 유형**:

| 유형 | 예시 | 인터랙션 |
|------|------|----------|
| 매출 요약 | "어제 매출 127만원, 전주 대비 +23%" | 탭 → Big Typography 풀스크린 |
| 넛지/알림 | "부가세 마감 D-3" | 스와이프 → 처리 플로우 |
| 인사이트 | "재료비가 평소보다 30% 높아요" | 탭 → 상세 분석 |
| 축하 | "이번주 최고 매출 달성! 🎉" | 폭죽 애니메이션 |
| 할 일 완료 | "영수증 3건 처리 완료 ✅" | 게이지 업 애니메이션 |
| 퀵액션 | "📸 영수증 찍기" | 카메라 즉시 실행 |

**피드 생성 로직**: 기존 hooks (useDashboardStats, useTransactions 등)의 데이터를 기반으로
김비서가 자연어 카드를 동적 생성.

---

### ④ 컨텍스트 칩 (동적)

보이스 버블 위에 수평 스크롤되는 칩들. 현재 상황에 따라 동적 변화:

| 상황 | 칩 구성 |
|------|---------|
| 아침 | [오늘 할 일] [어제 매출] [영수증] |
| 마감 임박 | [세금 처리] [서류 확인] [세무사 연결] |
| 매출 급등 | [상세 분석] [직원 칭찬] [리포트] |
| 첫 방문 | [김비서 소개] [연동하기] [둘러보기] |

---

### ⑤ 보이스 버블 (하단 네비 대체)

- 하단 네비게이션 바 **완전 제거**
- 하단 중앙에 보이스 버블 하나만 존재
- **탭 = 텍스트 입력 모드** (프라이버시 보호, 기본값)
- **길게 누름 = 음성 모드** (프리미엄 옵션)
- 음성 활성 시 파형(Waveform) UI로 인식 상태 피드백

---

### ⑥ 딥워크 오버레이

피드로 처리 불가능한 배치 작업을 위한 풀스크린 오버레이:
- "거래 정리해줘" → 거래 테이블 뷰 (기존 V1 재활용)
- "리포트 보여줘" → 리포트 뷰
- "직원 관리" → 직원 관리 뷰
- 하단 스와이프 또는 X 버튼으로 닫기

---

## 5. 기존 기능 접근 매핑 (탭 없이)

| 기존 탭 | V2 접근 방법 |
|---------|-------------|
| 대시보드 | → 피드 자체가 대시보드 |
| 매출/매입 | → "매출 보여줘" 음성 or 칩 → 풀스크린 카드 |
| 리포트 | → "이번 달 리포트" → 김비서가 피드에 카드 생성 |
| 자금 관리 | → "통장 잔고" → 즉시 응답 |
| 직원 관리 | → "직원 급여" → 딥워크 오버레이 |
| 설정/프로필 | → 기상도 옆 아바타 탭 → 슬라이드 패널 |

## 7. 허점 분석 및 보완

| 허점 | 위험 | 보완 |
|------|------|------|
| 탭 제로 → 발견 불가능성 | 뭘 할 수 있는지 모름 | 첫 진입 온보딩 피드 + 동적 컨텍스트 칩 |
| 피드 무한 스크롤 → 중요 항목 매몰 | 넛지 놓침 | 미처리 액션 Sticky 영역 분리 |
| 음성 의존 → 소음·프라이버시 | 매장에서 사용 불가 | 탭=텍스트(기본), 홀드=음성(옵션) |
| 딥워크 불가 → 배치 작업 한계 | 30건 거래 분류 곤란 | 딥워크 오버레이 (기존 V1 뷰 재활용) |
| 기상도 과도한 단순화 | 맑음인데 세금 폭탄 | 3축 상세 (매출·세금·증빙) |
